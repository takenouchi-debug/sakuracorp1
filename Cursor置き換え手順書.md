## 🎯 この手順書の目的

**「もぐら叩き」を完全に防ぐ**ために、Cursorで安全にファイルを置き換える方法を解説します。

---

## ⚠️ 最重要ルール

### ❌ やってはいけないこと
1. **CursorのAIに「修正して」と頼む**
   - → AIが勝手に色々変えて、もぐら叩き発生
2. **複数ファイルを一気に置き換える**
   - → どこで問題が起きたか分からなくなる
3. **バックアップを取らずに作業**
   - → 問題が起きても戻せない

### ✅ やるべきこと
1. **完璧なレシピ（修正済みファイル）を使う**
   - → Claudeが作った修正済みファイルをそのまま使う
2. **1ファイルずつ確実に置き換える**
   - → 1つ完了 → 確認 → 次へ
3. **必ずバックアップを取る**
   - → 問題があればすぐに戻せる

### ファイル1: index.html（トップページ）

#### 手順

1. **Cursorでindex.htmlを開く**
2. **全選択**: `Ctrl+A` (Windows) / `Cmd+A` (Mac)
3. **modified_files/index.htmlをメモ帳で開く**
4. **全選択してコピー**: `Ctrl+A` → `Ctrl+C`
5. **Cursorに戻って貼り付け**: `Ctrl+V`
6. **保存**: `Ctrl+S`

#### 確認

1. ブラウザでindex.htmlを開く
2. **桜吹雪が4秒間表示される**ことを確認
3. **キャッチコピーが「未来をトータルサポート」**になっていることを確認
4. **4つの事業の順序**が正しいことを確認

**✅ 問題なし** → 次のファイルへ  
**❌ 問題あり** → バックアップから復元してClaudeに相談

## ❌ トラブルシューティング

### 問題1: 桜吹雪が全ページで表示される

**原因**: main.jsが正しく置き換えられていない

**解決策**:
1. バックアップから復元
2. main.jsをもう一度置き換え
3. ブラウザのキャッシュをクリア（Ctrl+Shift+R）

---

### 問題2: ナビゲーションの順序がおかしい

**原因**: HTMLファイルが正しく置き換えられていない

**解決策**:
1. バックアップから復元
2. 該当ファイルをもう一度置き換え
3. 「修正指示書.md」で正しい順序を確認

---

### 問題3: レイアウトが崩れた

**原因**: ファイルの一部が欠けている可能性

**解決策**:
1. バックアップから復元
2. 該当ファイルをもう一度、**慎重に**置き換え
3. 全選択→コピー→貼り付けを確実に実行

## 🎓 プロのアドバイス：もぐら叩きを防ぐコツ

### 1. AIに余計なことをさせない

**❌ 悪い例**:
```
Cursorに「このファイルのナビゲーション順序を変えて」と頼む
→ AIが勝手に他の箇所も変える
→ もぐら叩き発生
```

**✅ 良い例**:
```
完璧なレシピ（修正済みファイル）を丸ごと貼り付ける
→ AIは何もしない
→ もぐら叩きゼロ
```

---

### 2. 1ファイルずつ確実に

**❌ 悪い例**:
```
10ファイルを一気に置き換える
→ どこかで問題発生
→ どのファイルが原因か分からない
```

**✅ 良い例**:
```
1ファイル置き換え → 確認 → 次へ
→ 問題があればすぐに特定できる
```

---

### 3. バックアップは命綱

**❌ 悪い例**:
```
バックアップなしで作業
→ 問題発生
→ 元に戻せない
→ 最初からやり直し
```

**✅ 良い例**:
```
作業前にバックアップ
→ 問題発生
→ バックアップから復元
→ 5分で元通り
```
